# SlintUI 内容、数据与本地化规范

本文定义组件中可复用的文案、数据格式、Locale、RTL、文本溢出和隐私规则。业务层决定内容含义，SlintUI 负责提供一致的展示与格式化接口。

## 1. 内容原则

- 用户视角：说明用户可以做什么、发生了什么以及下一步是什么。
- 准确完整：避免“失败”“错误”“无效”等没有上下文的孤立文案。
- 简洁直接：删除用户已经知道的事实，把重要信息放在前面。
- 术语一致：同一对象、操作、状态和页面标题使用同一名称。
- 正向尊重：不指责用户，不使用夸张、命令式或绝对化表达。

## 2. 操作文案

- Button 和 PopupMenuItem 使用动词或“动词 + 对象”，例如“保存”“删除文件”。
- 主操作表达任务结果，避免脱离上下文的“确定”“好的”“提交”。
- Cancel 表示放弃当前临时操作；Close 表示关闭展示，不应混用。
- 危险操作明确对象和影响，例如“删除 3 个文件”，不能只写“删除”。
- 打开后仍需继续填写或确认的菜单项可以使用省略号，例如“另存为…”。
- 页面标题、导航名称和打开该页面的操作名称保持一致。

## 3. 错误、警告与状态文案

错误文案包含三个部分，按场景省略不必要部分：

```text
发生了什么 + 影响范围 + 下一步操作
```

示例：

```text
无法保存设置。文件不可写。请检查目录权限后重试。
```

- 字段错误紧邻字段，说明正确格式或范围。
- 系统错误面向用户表达，不直接展示堆栈、内部异常或裸错误码；错误码可以作为可复制详情。
- Warning 表示可以继续但存在风险，必须说明继续的后果。
- Success 只在结果不明显时提示；界面已经清晰反映结果时不重复显示 Toast。
- 后台任务文案区分“已提交”“处理中”“已完成”“部分完成”“已取消”“失败”。

## 4. Label、帮助和 Placeholder

- Label 始终可见，Placeholder 只能提供示例或格式提示，不能代替 Label。
- 必填标记、单位、格式和范围在输入前可获得。
- 简短帮助放在输入下方；长说明通过帮助入口展示，但关键约束不能只放 Tooltip。
- Label 使用名词或简短问题，Switch 的 Label 明确开启后的状态。
- Tooltip 文案尽量是一句完整说明，不重复控件可见文字。

## 5. Locale 与 Direction

宿主为每个顶层窗口提供：

```text
locale: BCP 47 language tag，例如 zh-CN、en-US、ar-EG
direction: ltr | rtl
time-zone: IANA time zone，例如 Asia/Shanghai
numbering-system: 跟随 Locale，必要时显式覆盖
```

- 组件不读取操作系统全局 Locale 作为隐式回退；宿主未提供必要配置时返回明确错误。
- `direction` 控制布局、导航顺序和方向键的视觉语义，不通过字符串语言判断 RTL。
- 布局属性和文档使用 `start/end`、`before/after`，避免把 `left/right` 写入通用语义。
- Back、Forward、Chevron、Undo/Redo 等方向图标必须声明 RTL 是否镜像；播放、下载、搜索等非方向图标不镜像。
- 用户生成内容可以拥有与界面不同的文本方向，组件需支持局部方向。

当前 P0 实现状态：`Theme.locale` 与 `Theme.direction` 独立存在，Gallery 可切换中文、英文和阿拉伯语，并验证 RTL 文本对齐。稳定布局不会自动交换任意 `@children` 的结构顺序，方向图标也不会自动镜像；调用方必须按组件规格显式编排。完整自动结构镜像仍是待实现能力，不能用当前 Locale 展示替代该项验收。

## 6. 文本扩张与溢出

- Gallery 使用中文、英文、德语类长文本、RTL 和无空格长串验证。
- 普通操作文案至少预留英文文本相对中文 `1.5–2` 倍的宽度增长。
- 重要标题、错误、Label 和操作不能因为固定高度被裁剪。
- 单行省略只用于重复列表、表格和空间确实受限的区域，并提供完整内容访问方式。
- 多行正文优先换行；路径、URL、哈希和代码使用中间省略或可水平滚动策略。
- Tooltip 不能成为访问被截断内容的唯一键盘入口。
- 空字符串、Null、加载中和无权限是不同状态，不能统一显示为空白。

## 7. 数字与单位

- 数字格式跟随 Locale，包含小数点、千分位和数字系统。
- 表格数值右对齐并使用等宽数字；同一列保持相同精度。
- 单位与数值之间的空格、大小写和复数由格式化层处理。
- 百分比明确输入是 `0–1` 还是 `0–100`，组件 API 不猜测。
- 大数缩写仅用于概览，并提供完整精确值；重要金额和计量默认不缩写。
- 非数值、正负无穷和超出范围返回明确格式错误，不静默显示为 0。

推荐统一格式化接口：

```text
format-number(value, locale, options)
format-percent(value, locale, options)
format-currency(value, currency-code, locale, options)
format-measurement(value, unit, locale, options)
format-file-size(bytes, unit-system, locale)
```

## 8. 日期、时间与时区

- 存储和传输使用明确时区或 UTC；显示时转换为窗口环境的 `time-zone`。
- 界面格式跟随 Locale，不在组件内部固定 `YYYY-MM-DD`。
- 需要审计、日志或协作时优先显示绝对时间；强调新鲜度时可以显示相对时间，并提供绝对时间详情。
- 日期范围明确包含或排除结束时间；避免仅用连字符导致与负号混淆。
- 12/24 小时制跟随 Locale 或用户设置。
- 夏令时导致的不存在或重复时间必须由业务规则明确处理，不静默偏移。

推荐统一接口：

```text
format-date(instant, locale, time-zone, style)
format-time(instant, locale, time-zone, style)
format-date-time(instant, locale, time-zone, style)
format-relative-time(instant, now, locale, thresholds)
```

## 9. 空值、加载和未知状态

| 状态 | 推荐展示 |
|---|---|
| 有值 | 格式化后的真实值 |
| Null / 无数据 | `—` 或明确的“无数据” |
| 空字符串 | 按业务含义显示空，不自动等同 Null |
| 加载中 | Skeleton、Spinner 或“正在加载” |
| 加载失败 | 错误说明和重试入口 |
| 无权限 | “无权查看”及可用的权限说明 |
| 未知 | “未知”，不能显示为 0、False 或空白 |

表格同一列统一使用一种空值表示。复制数据时应复制原始值还是展示值，必须由组件 API 明确。

## 10. 隐私与脱敏

- 脱敏发生在业务数据进入组件之前，组件不根据字段名称猜测敏感级别。
- 脱敏结果必须与复制、导出、Tooltip 和可访问名称保持一致，不能通过其他通道泄露原值。
- 完整隐藏使用 `***` 或明确占位；部分隐藏保留足以识别但不足以还原的信息。
- 查看原值属于独立受控操作，需要权限检查、明确反馈和必要的审计。
- 密码、Token、身份证件、银行卡、联系方式和个人地址不得进入截图基线、日志或示例数据。

## 11. 标点与大小写

- 中文按钮、Label、菜单项和单句 Tooltip 末尾通常不加句号。
- 多句说明、段落和错误详情使用完整标点。
- 英文标题、Label、按钮使用 sentence case，产品专有名词除外。
- 谨慎使用感叹号，不使用全大写制造强调。
- 省略号统一使用 Unicode `…`，不使用三个句点替代。
- 快捷键、单位和协议名称保留标准大小写，例如 `Esc`、`GB`、`HTTP`。

## 12. 组件接口约束

- 组件接受业务文本或结构化值，不在内部拼接不可本地化的完整句子。
- 带数量的文案由本地化层处理复数规则，不用字符串拼接实现。
- 日期、金额和单位优先传入结构化数据，由统一格式化服务生成展示文本。
- 可访问名称默认使用可见文案；两者不一致时必须显式传入并说明原因。
- 组件内置文案集中在 Locale 资源中，缺少翻译时明确报错，不能静默切换到其他语言。
