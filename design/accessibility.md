# SlintUI 无障碍规范

SlintUI 以 WCAG 2.2 AA 作为通用验收基线，并结合 Windows、macOS、Linux 桌面控件惯例实现键盘、焦点、语义和系统辅助功能。WCAG 来源于 Web，但其中对比度、键盘、焦点、目标尺寸和文本缩放要求同样适合作为桌面组件库的最低标准。

## 1. 基本要求

- 所有核心功能可通过键盘完成，不能要求精确指针、拖动或 Hover。
- 信息不能只由颜色、位置、动画、声音或图标形状中的单一方式表达。
- 组件必须具有可访问名称、角色、状态和值；可见标签优先作为可访问名称。
- 主题、密度和文本缩放不能改变功能范围或操作顺序。
- 无障碍问题视为功能缺陷，不作为发布后的视觉优化处理。

## 2. 对比度

| 内容 | 最低要求 |
|---|---:|
| 普通正文和图像文字 | `4.5:1` |
| 大号文字（约 24px 常规或 19px 粗体以上） | `3:1` |
| 控件边界、状态图标和必要图形 | `3:1` |
| 焦点指示器与相邻颜色变化 | `3:1` |
| 正文、标题与背景 | 以 `7:1` AAA 为设计目标 |

- Disabled 控件不承担关键操作，但仍应可辨认并能读出其标签和禁用状态。
- Placeholder 不能代替 Label；其较低对比度不能承载唯一说明。
- 功能色必须配合图标、形状或文字，例如错误边框同时显示错误图标或说明。
- 透明色按最终合成背景计算对比度，不能只比较 Token 原始值。
- Gallery 为每个主题生成文本、图标、边框、焦点环和状态色的对比度报告。

## 3. 键盘与焦点

- Tab 顺序与视觉阅读顺序一致，不使用任意数值跳转顺序修补布局。
- 复合控件在 Tab 顺序中通常只占一个位置，内部按 [`interaction.md`](interaction.md) 使用方向键管理。
- 键盘焦点始终可见，不得仅通过轻微色差表示。
- 焦点不能被固定栏、Drawer、Popup 或滚动裁剪遮挡。
- Modal 打开后焦点进入 Modal 并被限制在其中；关闭后返回触发器或逻辑后继对象。
- 删除当前聚焦对象后，必须把焦点移动到相邻项、父容器或下一任务入口。
- Disabled 元素默认不进入 Tab 顺序；当可发现性依赖其存在时，可以保留焦点但必须禁止激活并说明原因。

## 4. 命中区域与输入方式

- 所有独立指针目标至少为 `24 × 24px`；相邻目标过密时必须扩大目标或增加间距。
- `compact` 密度允许可视控件为 24px，但透明命中区域不得相互重叠。
- `comfortable` 密度的主要目标至少为 `44 × 44px`。
- 拖动、双击、右键和触摸板手势必须存在按钮、菜单或键盘替代。
- Pointer 按下后拖出目标再释放不得误触发；取消操作不能造成数据变化。

## 5. 文本与缩放

- 支持至少 200% 文本缩放，不丢失内容和功能。
- 文本放大后允许控件增高、标签换行和页面滚动；禁止缩小字体维持原布局。
- 正文、标签、帮助、错误和快捷键提示使用真实文本，不使用图片文字。
- 长文本截断时提供可访问的完整内容；重要错误和操作说明不得只存在于 Tooltip。
- 行高、段落间距和字间距不能被组件写死到导致系统文本设置失效。

## 6. 可访问语义

每个组件规格必须记录：

| 项目 | 示例 |
|---|---|
| Role | button、checkbox、dialog、list、tree |
| Name | 可见 Label、标题或明确的替代名称 |
| Description | 帮助说明、格式要求、危险影响 |
| State | disabled、checked、selected、expanded、busy、invalid |
| Value | 当前值、范围、单位、位置 |
| Relationship | Label 与字段、错误与字段、标题与 Dialog |

- 图标按钮必须有不依赖 Tooltip 的可访问名称。
- 状态变化、后台任务和校验结果必须通过平台可访问通知机制播报，且避免重复播报。
- 列表、树、表格必须暴露项数、当前位置、选中状态和层级关系。
- 加载区域设置 Busy 状态；加载结束后只在结果对当前任务重要时播报。

## 7. 错误与表单

- Label 始终可见，并与输入控件建立语义关联。
- 必填、格式、范围和单位在输入前可获得，不只在失败后说明。
- 提交失败后先展示错误摘要，再将焦点移到第一个可修复错误。
- 错误文案说明问题和修复方式，不只显示“无效”或错误代码。
- Error 不能自动清空输入；用户修复后及时更新状态。
- 法律、财务、删除和重要数据提交应允许检查、确认或撤销。

## 8. 动效、闪烁与时间限制

- 跟随系统减少动效设置；关闭动效后仍保留状态变化的非动画反馈。
- 不使用快速闪烁、持续脉冲或大面积视差表达必要信息。
- 自动关闭的信息在 Hover 或聚焦时暂停；重要内容不得仅依赖超时阅读。
- 倒计时、会话过期和限时操作必须提前提示，并在安全允许时提供延长时间。
- 加载动画不得使屏幕阅读器重复播报同一状态。

## 9. 高对比度和系统设置

- 高对比度主题使用系统语义色时，不再依赖透明度和阴影表达边界。
- Focus、Selected、Error 和 Disabled 必须有可同时辨认的边框、图形或文字差异。
- 跟随操作系统主题、高对比度、文本缩放和减少动效设置；用户在应用中的明确选择可以覆盖主题和密度，但不能关闭必要的无障碍能力。
- 系统颜色或语义 API 不可用时返回明确能力状态，不能静默使用未经验证的近似主题。

## 10. 测试矩阵

组件达到“已完成”前至少验证：

- 仅键盘完整操作。
- 100%、125%、150%、200% 系统缩放和 200% 文本缩放。
- 浅色、深色、高对比度以及减少动效。
- 中文、英文、长文本和 RTL。
- 屏幕阅读器或平台无障碍检查工具读取名称、角色、状态和值。

当前实现已声明 Slint 公开的名称、角色、checked、enabled、read-only、值与 live-region 属性，并提供键盘焦点和减少动效场景。Slint 1.17.1 公开能力不能表达 checkbox mixed 和字段 required 状态；Narrator、NVDA、VoiceOver、Orca 的实际播报仍需三平台真机验证，相关任务统一记录在 [`TODO.md`](../TODO.md)，当前不标记为已经通过。
- 焦点不丢失、不被遮挡，Modal 和 Popup 关闭后正确恢复。
- 文本、图标、边框和焦点的自动对比度检查。

平台真机验证基线：

```text
Windows: Narrator、键盘、高对比度、文本缩放
macOS:   VoiceOver、Full Keyboard Access、Increase Contrast
Linux:   Orca、键盘、高对比度主题（桌面环境允许时）
```
